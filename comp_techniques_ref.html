Title
Develop algorithm, draw flow chart and write a program to implement: (a) if-else and function — application in Intermediate Value Theorem. (b) 2D Array formation — application in matrix data entry, transposition and printing matrix.
Aim
To develop algorithm, draw flow chart and write a program to implement if-else and functions in application of Intermediate Value Theorem, and 2D Array formations in matrix data entry, transposition and printing matrix.
Theory
Intermediate Value Theorem: If f(x) is continuous in interval [a,b] and f(a) and f(b) have different signs, then the equation f(x) = 0 has at least one root between x = a and x = b.
Matrix: A matrix is a rectangular arrangement of numbers (real or complex) enclosed by [ ] or ( ) or || ||. In compact form, the above matrix is represented by [aᵢⱼ] m×n or A = [aᵢⱼ] where i is the row index and j is the column index.
Transpose: If A = [aᵢⱼ] is an m×n matrix, the matrix obtained by interchanging its rows and columns is called the transpose of A, denoted A' or Aᵀ. Then A' = [aⱼᵢ] of order n×m.
Types of Matrices
  • Row matrix: A matrix having only one row and any number of columns.
  • Column matrix: A matrix having only one column and any number of rows.
  • Square matrix: A matrix of order m×n such that m = n.
  • Rectangular matrix: A matrix of order m×n such that m ≠ n.
  • Null/Zero matrix: All elements are zero, i.e. aᵢⱼ = 0 ∀ i,j.
  • Diagonal matrix: Square matrix A = [aᵢⱼ] where aᵢⱼ = 0 for i ≠ j.
  • Identity/Unit matrix: Diagonal matrix where all diagonal elements = 1.
  • Singular matrix: Square matrix whose determinant = 0, i.e. |A| = 0. Otherwise non-singular.
Algorithm — IVT
1Read the interval [a, b].
2Calculate f(a) and f(b).
3If f(a) × f(b) < 0, then root lies in the interval [a,b]. Else root does not lie in the interval.
4Display result and stop.
Algorithm — Matrix Transpose
1Enter the number of rows R and columns C.
2Read the coefficients of matrix row wise.
3Display the matrix on screen.
4Calculate the transpose of matrix.
5Display the transposed matrix and stop.
Flowchart — IVT
START → [Input Interval [a,b]] → [Calculate f(a) and f(b)] → <f(a) × f(b) < 0?> YES → [Root lies in the interval] NO → [Root does NOT lie in the interval] → [Display result] → STOP
Flowchart — Matrix Transpose
START → [Read no. of rows and columns] → [Read the coefficient of matrix] → [Display the matrix on screen] → [Calculate the transpose] → [Display the transposed matrix] → STOP

Experiment No. 2
Title
Solution of transcendental/polynomial equation using Bisection method.
Aim
To solve given transcendental equation f(x) = cos(x) − xe^x using Bisection method and write a Python program for bisection method.
Theory
Bisection method is the simplest among all numerical schemes to solve transcendental equations. The sequence of approximations always converges to the root for any f(x) which is continuous in the interval. This scheme is based on the Intermediate Value Theorem for continuous functions, which states that if f(x) is continuous in the interval [a,b] and f(a), f(b) have different signs, then the equation f(x) = 0 has at least one root between x = a and x = b.
Consider a transcendental equation f(x) = 0 which has a root in the interval [a,b] and f(a)×f(b) < 0. Bisection scheme computes the root by repeatedly halving the interval [a,b]. Starting with c = (a+b)/2, the interval is replaced either with [c,b] or with [a,c] depending on the sign of f(a)×f(c) and f(c)×f(b).
c = (a + b) / 2 Number of iterations needed: n ≥ [log(b−a) − log(e)] / log(2)
Advantages
  • Simple method.
  • Can be used for both polynomial and transcendental equations.
  • Method always converges as it is a bracketed method.
Disadvantages
  • May fail when the function is tangent to the axis and does not cross f(x) = 0.
  • This method is generally more slowly convergent than most other methods for functions f(x) that have a continuous derivative.
  • Bisection method requires large number of iterations.
Algorithm
1Read interval [a,b] and permissible error e.
2Calculate number of iterations needed: n ≥ [log(b−a) − log(e)] / log(2).
3Find next approximation c = (a+b)/2.
4Calculate f(a), f(b) and f(c).
5If f(a)×f(c) < 0, new interval is [a,c], replace b by c. Else if f(c)×f(b) < 0, new interval is [c,b], replace a by c.
6Repeat step 3 to 6 until number of iterations are reached.
7Display approximate value of root and stop.
Flowchart
START → [Input interval [a,b] and iterations n] → [c = (a+b)/2] → [Calculate f(a), f(b), f(c)] → <f(a) × f(b) < 0?> YES → <f(a) × f(c) < 0?> YES → b = c → loop back NO → a = c → loop back NO → [Root not in interval] → [n = n−1] → <n ≤ 0?> YES → Root = c → [Display root] → STOP NO → loop back to c = (a+b)/2
Conclusion
From this experiment we have studied to write a program for solution of transcendental equation using Bisection Method and to develop algorithm and draw flow chart.

Title
Solution of transcendental equation using Newton-Raphson method.
Aim
To solve the given transcendental equation using Newton-Raphson method and write a program for it.
Theory
Let x₀ be an approximate root of equation f(x) = 0. If x₁ = x₀ + η be the exact root, then f(x₁) = 0. Expanding f(x₀ + η) by Taylor's series and since η is small, neglecting η² and higher powers of η we get f(x₀) + ηf'(x₀) = 0.
η = −f(x₀) / f'(x₀) x₁ = x₀ − f(x₀)/f'(x₀) General: xₙ₊₁ = xₙ − f(xₙ)/f'(xₙ) (n = 0, 1, 2, 3 ...)
This is known as the Newton-Raphson formula or Newton's iteration formula.
Observations
  • Obs 1: Newton's method is useful in cases of large value of f'(x), i.e. when the graph of f(x) while crossing the x-axis is nearly vertical. If f'(x) is small near the root, computation is slow or may not be possible.
  • Obs 2: Geometrical interpretation — the method consists of replacing the part of the curve between point A₀ and the x-axis by the tangent to the curve at A₀.
  • Obs 3: Newton's method is generally used to improve the result obtained by other methods. It is applicable to solution of both algebraic and transcendental equations.
  • Obs 4: Newton's formula converges provided the initial approximation x₀ is chosen sufficiently close to the root. A bad initial choice will lead to an endless cycle.
Algorithm
1Start.
2Read the values of x₀ and n (number of iterations).
3Calculate f(x) and f'(x).
4Calculate next approximation: x₁ = x₀ − f(x₀)/f'(x₀).
5Check whether number of iterations has reached. Yes → stop. No → go to step 3, make x₁ = x₀.
6Display the root and stop.
Flowchart
START → [Input initial approximation x₀ and iterations n] → [Calculate f(x), f'(x)] → [i = 1] → [Calculate x₁ = x₀ − f(x₀)/f'(x₀)] → [i = i + 1] → <i ≤ n+1?> YES → x₀ = x₁ → loop back NO → [Display root = x₀] → STOP
Conclusion
From the experiment we learnt to solve the transcendental equation using Newton-Raphson method and to develop program and flowchart for it.

yₚ = y₀ + pΔy₀ + p(p−1)/2! × Δ²y₀ + p(p−1)(p−2)/3! × Δ³y₀ + ... where p = (x − x₀) / h and h = x₁ − x₀
If y = f(x) is a polynomial of nth degree then Δⁿy and higher differences will be zero.

Title
Program for interpolation using Lagrange's interpolation.
Aim
To write a program for interpolation using Lagrange's interpolation.
Theory
Let f(x₀), f(x₁), ..., f(xₙ) be the (n+1) values of the function y = f(x) corresponding to arguments x₀, x₁, x₂, ..., xₙ, not necessarily equally spaced. It is assumed that the function f(x) is a polynomial in x of degree n.
f(x) = [(x−x₁)(x−x₂)...(x−xₙ)] / [(x₀−x₁)(x₀−x₂)...(x₀−xₙ)] × f(x₀) + [(x−x₀)(x−x₂)...(x−xₙ)] / [(x₁−x₀)(x₁−x₂)...(x₁−xₙ)] × f(x₁) + ... + [(x−x₀)(x−x₁)...(x−xₙ₋₁)] / [(xₙ−x₀)(xₙ−x₁)...(xₙ−xₙ₋₁)] × f(xₙ)
This is Lagrange's Interpolation Formula. Works for both equally and unequally spaced data.
Algorithm
1Enter the maximum number of data points.
2Read all the data points and read values of x = xr at which y is to be interpolated.
3For each j, calculate numerator = product of (xr−x[i]) for i≠j, and denominator = product of (x[j]−x[i]) for i≠j. Add (num/den)×y[j] to fy.
4The interpolated value at x = xr is equal to fy.
5Display the value and stop.
Flowchart
START → [Read max number of data points] → [Read actual values of data (x and y)] → [Read values of xr at which yr is to be calculated] → [Calculate value of yr using Lagrange's interpolation formula] → [Display value of xr and yr] → STOP

Title
Solution of Numerical Integration using Simpson's 1/3 Rule.
Aim
Write a program to evaluate integral of given function by dividing the integral in parts using Simpson's 1/3 rule.
Theory
Trapezoidal rule approximates the graph between two adjacent points by the straight line joining them. Against this, Simpson's Rule makes use of the parabola joining three adjacent points to approximate the graph. A parabola can be drawn through any three non-collinear points. The Simpson formula is arrived at by dividing the given interval [a,b] into 'n' sub-intervals of equal length where n must be an even integer.
I = h/3 × [(y₀+yₙ) + 4(y₁+y₃+...+yₙ₋₁) + 2(y₂+y₄+...+yₙ₋₂)] = (1/3) × h × [first+last + 4×(odd ordinates) + 2×(even ordinates)] This formula is also called the Parabolic formula. n MUST be even.
Algorithm
1Enter initial values: upper limit and lower limit.
2Read value of spacing h or number of intervals n. Let n = (xₙ−x₀)/h.
3Calculate the values of y for corresponding values of x.
4Calculate I = h/3 × [(y₀+yₙ) + 4(odd terms) + 2(even terms)].
5Display the result.
6Stop.
Flowchart
START → [Enter lower limit, upper limit and number of intervals] → [Calculate step size h = (upper−lower)/n] → [Calculate y₀, y₁, y₂, ..., yₙ] → [Calculate I using Simpson's 1/3 Rule formula] → [Print output] → STOP
Conclusion
In this experiment, we learned Simpson's 1/3 rule, flowchart and algorithm for Simpson's 1/3 rule.
Experiment No. 7
Title
Solution of first order ODE using 4th order Runge-Kutta method.
Aim
Write a program to evaluate dy/dx = f(x,y), find y(x) using 4th order Runge-Kutta method.
Theory
Taylor series method involves finding out the higher order derivatives of y numerically. RK method was introduced to avoid calculations of higher order derivatives. It requires calculations only at some selected points. In the 4th order RK method, we calculate the increment K corresponding to an increment h in x.
K₁ = h·f(x₀, y₀) ← full step slope K₂ = h·f(x₀+h/2, y₀+K₁/2) ← midpoint slope K₃ = h·f(x₀+h/2, y₀+K₂/2) ← better midpoint slope K₄ = h·f(x₀+h, y₀+K₃) ← endpoint slope K = (1/6)(K₁ + 2K₂ + 2K₃ + K₄) ← weighted average y₁ = y₀ + K
Algorithm
1Enter the initial values x₀, y₀ and step size h.
2Read the values of x at which y is to be found.
3Calculate K₁, K₂, K₃, K₄ at current x₀, y₀.
4Calculate K = (1/6)(K₁ + 2K₂ + 2K₃ + K₄).
5Calculate y₁ = y₀ + K and x₁ = x₀ + h.
6Repeat step 3 to 5 till x₀ reaches xr (the required x value).
7Display the result and stop.
Flowchart
START → [Read x₀, y₀, xr, h] → [Calculate K₁, K₂, K₃, K₄] → [K = (K₁+2K₂+2K₃+K₄)/6] → [y₁ = y₀ + K, x₁ = x₀ + h] → <x₀ < xr?> YES → x₀ = x₁, y₀ = y₁ → loop back NO → [Display x₁, y₁] → STOP
Conclusion
In this experiment we studied to solve numerical ODE by developing program and algorithm using Runge-Kutta 4th order method.

Aim
Find the solution to the following system of equations using Jacobi method: 20x₁ + x₂ − 2x₃ = 17, 3x₁ + 20x₂ − x₃ = −18, 2x₁ − 3x₂ + 20x₃ = 25. Take initial approximation x₁ = x₂ = x₃ = 0, show 3 iterations.
Theory
In certain cases such as when a system of equations is large, iterative methods of solving equations are more advantageous. Elimination methods such as Gaussian elimination are prone to large round-off errors for a large set of equations. Iterative methods such as the Jacobi method give the user control of the round-off error.
The Jacobi method requires that the diagonal elements are non-zero. Each equation is rewritten for its diagonal variable. Starting with initial approximation x₀, y₀, z₀ and substituting on right hand side gives first approximations:
x = (1/a₁₁)(d₁ − a₁₂y − a₁₃z) y = (1/a₂₂)(d₂ − a₂₁x − a₂₃z) z = (1/a₃₃)(d₃ − a₃₁x − a₃₂y) General: x^(k+1) = (1/a₁₁)(d₁ − a₁₂y^k − a₁₃z^k) Rearrange rule: a·x + b·y + c·z = d → x = (d − b·y − c·z) / a
Algorithm
1Read the number of variables n in the equations.
2Read the coefficients of equations aᵢⱼ for i=1 to n, j=1 to n and dᵢ.
3Arrange equations such that diagonal elements are dominant and represent system in iterative form.
4Take initial values of approximations for the number of variables.
5Execute the iterative equations with values of variables from initial values.
6Repeat step 5 until number of iterations or required accuracy is achieved.
7Display the values and STOP.
Flowchart
START → [Read number of variables] → [Read coefficient of variables] → [Arrange equations so diagonal elements are dominant] → [Take initial values of approximations] → [Calculate x₁=f1(x0,y0,z0), y₁=f2(...), z₁=f3(...)] → <Desired iterations reached?> YES → [Display the results] → STOP NO → x₀=x₁, y₀=y₁, z₀=z₁ → loop back
